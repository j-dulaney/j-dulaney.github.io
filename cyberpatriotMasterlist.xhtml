Before you start anything, read the “readme” in the image. This will dictate what you need to do and what you do not need to do. There are also forensics questions you need to read before you make any changes. 9 times out of 10 the forensics questions are also a scored problem for you to fix. Once you know what you need to do and not to do, continue.


-Enable Firewall

 Ubuntu’s Firewall is not enabled by default. You can enable it using ufw --enable or by  installing the GUI version of the firewall called “GUFW”
  sudo apt-get install gufw

 If you use the GUI version, just open it once installed and switch the slider to “on” or “active”.

  - Open terminal

                - sudo apt-get install gufw
                - Open gufw
                - Move slider to “on” or “active”

-Compare users on PC
 The readme will list users that belong on the PC and the privelidges they have. This includes         being an administrator or a standard user. Compare the users in the list (top right corner of         Ubuntu or start menu, User Accounts). If a user belongs but is in the wrong group, click the         lock button in the manager and type password to unlock and make changes. Also users are         found on the system that are not supposed to be on at all. Remove these, but remember to         check the forensics questions as I have had bad accounts be a question in prior years.

  - Start or top right
                - User Accounts

  - Find accounts that belong or do not belong

                - Verify accounts that belong are in the right priviledge                                 group (administrator, standard)

-Disable guest account
 Guest account is enabled in most images. This is a hazard and has been scored in the past. This  is a side-account that can be used to access the system and is a vulnerability in the sense of  physical access. To disable, you must locate the configuration file for “lightdm”, the login  package for Ubuntu and change guest-account to false.
  - Open terminal
                - sudo gedit /etc/lightdm/lightdm.conf
                - Add the line “allow-guest=false” to the end of the file                         (no quotes)
                - Save
                - Restart lightdm (restart PC)









- Disable autologin
 You can do this via the User Accounts manager or in the previous lightdm.conf file. Change it to false in either way and you will no longer autologin as that user. In recent years CyberPatriot has not required this to be a problem for points as it adds convenience to competitors, so read the footnotes of the readme.
 - Open terminal
        - sudo -H gedit /etc/lightdm/lightdm.conf
        - Remove autologin-user line(s)

 - Save

 - Restart lightdm if you want to be sure it registers the change

 

- Change passwords
 To change passwords to an account, type sudo passwd <username> followed by the requested  passwords. It is good practice to change passwords, but it is a better idea to follow the practice  of changing cracklib rules and pam.d configuration then locking the account. This enforces the  new policies you add by making the users accounts locked, requiring a new password following  policies.1
  Changing passwords
                - sudo passwd <username>
                - Type new password twice
                Changing policies and locking accounts
                - Open terminal
                - sudo gedit /etc/pam.d/common-password
                - Add “remember=5” to the end of line that has                                 “pam_unix.so” in it
                - Add “minlen=8” to the end of line that has “pam_unix.so”                 in it

  - Save
  - sudo gedit /etc/login.defs
  - Find in file (CTRL+F) “PASS_MAX_AGE”
  - Change PASS_MAX_DAYS to 90
  - Change PASS_MIN_DAYS to 10
  - Change PASS_WARN_AGE to 7
  - Save

 








- Setting account lockout policy
        You know when you mistype your password or pin for your phone and it locks you out for a set         amount of time? This discourages brute force attacks. Brute force attacks are when a malicious         individual tries every possible combination of letters, numbers, and characters to try and get the         right password. This is inefficient but with some passwords it is a severe vulnerability. 1234         would be vulnerable as it is an incrementation of simple counting. Regardless, it is good         practice to lockout an account the same way your phone can to you.
                - Open terminal
                - sudo gedit /etc/pam.d/common-auth
                - Add the following line to the end of the file
                auth required pam_tally.so deny=5 onerr=fail                                 unlock_time=18002
                - Save

- Enable automatic updates
        Automatic updates is a cornerstone of keeping a system secure. This ensures that when a fix is         released for a potentially devestating bug, the system is secured as soon as it is fixed.
                - Top right cog
                - System Settings
                - Software and Updates
                - Updates
                - All checkboxes should be checked
                - “Automatically check for updates” should be set to daily
                - “When there are security updates” should be set to                         download         and install automatically
                - “When there are other updates” should be set to display                 daily

 

- Check for updates
Checking for updates is an easy two lines of commands in terminal. This will refresh the cache, or stored information about what updates are available, and then pull them from repositories to install on your system.
                - sudo apt-get update
                - sudo apt-get upgrade
You may be prompted to press “y” to continue in the process

- Disable SSH login as root
Most competition images (but not all) require the use of SSH from authorized users. This following action disables ssh login as root. This ensures if a user wants to do something as root from a remote location, they do it under their name or not at all.
        - sudo gedit /etc/ssh/sshd.config
        - Change PermitRootLogin to no

 

- Make Firefox default browser & update it

                Updating Firefox should but is not always done through apt-get update & upgrade. Other methods include purging Firefox and reinstalling it, or using the built in installer. Check the current Firefox version here (https://www.mozilla.org/en-US/firefox/releases/)
        To update through apt-get
                - sudo apt-get update
                - sudo apt-get upgrade
        To update through uninstall and reinstall
                - sudo apt-get purge firefox
                - sudo apt-get install firefox
        To update through built-in updater
                - Find the “about” menu, usually located on the three tabs in the top right corner of the browser bar
                - Question mark
                - About Firefox

  - Check version (It should check for updates here, if not   and it is an outdated version, try updating through apt-get   or uninstall/reinstall)

 


- Check dpkg for potentially malicious installed packages
        dpkg is the packaging application for Debian based systems. Sifting through installed packages can find installed applications you might not find elsewhere.
                -dpkg –-list

  *You can also output the contents of this command to a file, for somewhat better reading by adding “> results.txt” to the end of the command.

                Sorting through this list can be tedious, but searching for phrases like “crack” or “password” sometimes yields relevant results
                -dpkg –-list | grep password
               

- Check group file for inconsistancies

        Checking the group file for inconsistancies can be fruitful. You can find users in a group that they do not belong. Careful, however, as there are several groups on systems by default for programs.

  -cat /etc/group
  *Like dpkg, you can output the command to a file for preference

 

- Lock all user accounts setting password reset in motion
        Locking accounts acts as if the password reset countdown is set to 0, making the user set a new password on their first login
 passwd -l username

 

- Removing a package

        sudo apt-get remove packagename
 If it is not removable via apt-get, use “dpkg –remove packagename” instead
